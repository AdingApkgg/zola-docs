使用

你应该遵循关于安装 Zola 主题的 官方文档。

我建议将主题添加为 git 子模块：

cd my-zola-website
git submodule add -b main git@github.com:tchartron/blow.git themes/blow

编辑你的 config.toml 文件中使用的主题

# 使用的站点主题。
theme = "blow"

然后编辑你的 config.toml 文件以覆盖主题的值：

[extra]
enable_search = true
enable_sidebar = true
enable_adsense = true
enable_multilingue = true

[extra.lang]

    ] },

    ] },

语法高亮

Blow 利用了 Zola 代码高亮功能。 它支持根据用户选择的主题（暗色 / 亮色）设置不同的配色方案。 为了使用它，你应该在 这里 提供的列表中选择你想用于亮色和暗色主题的配色方案，并像此示例一样编辑你的 config.toml 文件：

highlight_theme = "css"

highlight_themes_css = [
  { theme = "ayu-dark", filename = "syntax-dark.css" },
  { theme = "ayu-light", filename = "syntax-light.css" },
]

自定义页脚内容

要覆盖默认页脚（版权声明），请按照 Zola 文档 中的描述，通过在你的 templates 目录中创建一个包含以下内容的 layout.html 来扩展主题的 layout.html 模板：

{%/* extends "blow/templates/layout.html" */%}

{%/* block content_footer */%}
Here is my own footer with a <a href="http://example.com">link</a>.
{%/* endblock */%}

特性

  • 暗色/亮色模式（根据所选主题进行语法高亮）
  • 可自定义的导航栏链接
  • 标签和分类分类法
  • 支持 Command + K 快捷键的搜索功能
  • 社交链接（github, gitlab, twitter, linkedin, email）
  • 使用 cssnano 的 Postcss 构建过程（以及 tailwindcss tree shaking 以减小最终包大小）
  • 使用 minification 的 Uglifyjs 构建过程
  • 部署到 Github Pages 的示例脚本
  • 分页
  • 带有部分链接的侧边菜单
  • 目录（2 级和当前查看部分高亮）
  • 多语言
  • 404
  • 移动端响应式
  • Favicon
  • Adsense
